# Update Changelog Skill

**Purpose**: 프로젝트 변경 이력을 CHANGELOG.md에 자동으로 추가

## Description

이 skill은 현재 대화에서 수행한 작업들을 `CHANGELOG.md`에 자동으로 기록합니다.

## Instructions

사용자가 `/update-changelog` 명령을 실행하면:

1. 현재 대화에서 수행한 주요 변경 사항을 식별합니다
2. `CHANGELOG.md`의 최신 날짜 섹션을 확인합니다
3. 오늘 날짜와 일치하면 해당 섹션에 추가, 아니면 새 날짜 섹션을 생성합니다
4. 변경 사항을 카테고리별로 정리하여 추가합니다

## Target File

**변경이력 파일**: `CHANGELOG.md`

## Changelog Format

```markdown
---

## {YYYY-MM-DD}

### {카테고리 제목}
- {변경 내용 1}
- {변경 내용 2}
- {관련 링크가 있으면 포함}

### {다른 카테고리 제목}
- {변경 내용}

---
```

## 카테고리 분류 기준

| 카테고리 | 설명 | 예시 |
|---------|------|------|
| **모델 검증** | 특정 모델에 대한 검증 작업 | "Solar-Open-100B 검증 완료" |
| **문서 추가/수정** | 문서 작성 또는 업데이트 | "튜토리얼 Q11 추가" |
| **방법론 개선** | 검증 방법론 관련 변경 | "검증 방법론 한계 섹션 추가" |
| **프로젝트 구조** | 파일/폴더 구조 변경 | "skills 폴더 생성" |
| **버그 수정** | 오류 수정 | "링크 오류 수정" |

## Process

1. `CHANGELOG.md` 파일을 읽어서 현재 구조를 파악합니다
2. 오늘 날짜(YYYY-MM-DD)의 섹션이 있는지 확인합니다
3. 현재 대화에서 수행한 작업을 식별합니다:
   - 파일 생성/수정 내역
   - 검증 수행 내역
   - 문서 업데이트 내역
4. 변경 사항을 카테고리별로 그룹화합니다
5. 적절한 위치에 변경 내용을 삽입합니다:
   - 오늘 날짜 섹션이 있으면: 해당 섹션의 마지막에 추가
   - 오늘 날짜 섹션이 없으면: `---` 구분선 바로 아래에 새 섹션 생성

## 변경 내용 작성 규칙

1. **간결하게**: 한 줄로 핵심만 기술
2. **링크 포함**: 관련 외부 리소스가 있으면 마크다운 링크로 포함
3. **구체적으로**: "문서 수정" 대신 "Q11 방법론 비판 섹션 추가"
4. **참조 추가**: 논문, GitHub 이슈 등 출처가 있으면 명시

## Example Usage

```
User: /update-changelog
```

### 예상 출력 예시

```markdown
## 2026-01-05

### 방법론 개선
- 검증 방법론 한계와 학술 연구 결과 섹션 추가
- [arXiv:2502.00706v1](https://arxiv.org/abs/2502.00706) Black-box Output Similarity Testing 방법론 참조
- Yi-Llama 논쟁 사례 연구 추가

### 문서 추가
- 튜토리얼 Q11: 방법론 비판에 대한 답변 추가
- README.md에 방법론 한계 섹션 추가

### 프로젝트 구조
- `.claude/skills/update-changelog.md` skill 추가
```

## Notes

- 중복 기록 방지: 이미 CHANGELOG에 있는 내용은 다시 추가하지 않습니다
- 날짜 기준: 시스템 날짜 기준으로 섹션을 구분합니다
- 순서: 최신 날짜가 위에 오도록 유지합니다
